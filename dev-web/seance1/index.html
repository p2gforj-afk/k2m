Qu'est ce que le HTML?

HTML (HyperText Markup Language) n'est pas un langage de programmation : c'est un langage de balisage qui sert à indiquer au navigateur comment structurer les pages web visitées. Il peut être aussi compliqué ou aussi simple que le développeur web souhaite qu'il soit. Le HTML se compose d'une série d'éléments avec lesquels vous pouvez encadrer, envelopper ou baliser différentes parties du contenu pour les faire apparaître ou agir d'une certaine manière. Des balises encadrantes peuvent transformer une petite partie de contenu en un lien vers une autre page sur le Web, mettre des mots en italique, etc.

Éléments ligne et éléments blocs

Il existe deux catégories importantes d'éléments en HTML que vous devez connaître : les éléments de niveau bloc et les éléments en ligne.

Les éléments de niveau bloc
forment un bloc visible sur une page — ils apparaissent sur une nouvelle ligne quel que soit le contenu précédant et tout contenu qui les suit apparaît également sur une nouvelle ligne. Les éléments de niveau bloc sont souvent des éléments structurels de la page et représentent, par exemple, des paragraphes, des listes, des menus de navigation, des pieds de page, etc. Un élément de niveau bloc ne peut pas être imbriqué dans un élément en ligne, mais il peut être imbriqué dans un autre élément de niveau bloc.
Les éléments en ligne
sont contenus dans des éléments de niveau bloc. Ils entourent seulement des petites parties du contenu du document, ni des paragraphes entiers, ni des regroupements de contenu. Un élément en ligne ne fait pas apparaître une nouvelle ligne dans le document. Il apparaît généralement dans un paragraphe de texte, par exemple un élément <a> (hyperlien) ou des éléments de mise en évidence tels que <em> ou <strong>.

Tous les éléments ne suivent pas le modèle ci-dessus d'ouverture de balise, puis contenu, puis fermeture de balise. Certains éléments ne sont composés que d'une balise. Ils servent généralement à insérer / incorporer quelque chose dans le document à l'endroit où ils sont mis. Par exemple, l'élément <img /> ou <img> insère une image dans une page à l'endroit où il est placé (la balise auto-fermante <img /> est à privilégier)

Les éléments peuvent avoir des attributs

Les attributs contiennent des informations supplémentaires sur l'élément sans qu'elles n'apparaissent dans le contenu réel. Par exemple :

<div class="divClass1">Ceci est un bloc div de classe 1</div>

Ici, l'attribut class vous permet de donner à l'élément un nom d'identification qui peut ensuite être utilisé pour cibler l'élément afin de lui attribuer un style CSS ou un comportement particulier, par exemple.

Les entités nommées : inclure des caractères spéciaux

En HTML, les caractères <, >,",' et & sont des caractères spéciaux. Ils font partie de la syntaxe HTML elle-même, alors comment inclure un de ces caractères dans du texte, par exemple si vous voulez vraiment utiliser une esperluette (&) ou un signe inférieur (<), qui ne soit pas interprété en tant que code comme les navigateurs pourraient le faire ?

Nous devons utiliser les références des caractères — des codes spéciaux qui représentent des caractères et peuvent être utilisés dans ces circonstances exactes. Chaque référence de caractère est démarrée avec une esperluette (&), et se termine par un point-virgule (;).

  • < : &lt;
  • > : &gt;
  • " : &quot;
  • ' : &apos;
  • & : &amp;

Les Hyperliens

Les Hyperliens sont ce qui fait du Web une toile.

Ils nous permettent de lier nos documents à n'importe quel autre document (ou autre ressource) voulu ; nous pouvons faire des liens vers des parties précises de documents et rendre des applications disponibles à une simple adresse web.

Un lien élémentaire se crée en intégrant le texte (ou tout autre contenu) que vous voulez transformer en lien dans un élément <:a> et en lui affectant un attribut href (également connu comme étant une Hypertext Reference) contenant l'adresse web vers laquelle vous voulez que le lien pointe.

L'autre attribut qu'il est possible d'ajouter à un lien est title ; il est destiné à contenir des informations utiles supplémentaires à propos du lien, comme le type d'informations contenues dans la page ou ce qu'il faut savoir.

Les listes

Les listes sont aussi très répandues sur le Web, nous allons nous intéresser aux trois différents types de liste.

les listes non-ordonnées
Les listes non-ordonnées sont utilisées pour représenter des listes d'éléments pour lesquelles l'ordre n'a pas d'importance (comme une liste de courses). Les listes non-ordonnées débutent par un élément <ul> (unordered list) qui enveloppe tous les éléments de la liste.
les listes ordonnées
Les listes ordonnées permettent de représenter des listes dans lesquelles l'ordre des éléments a de l'importance (comme une recette de cuisine). Les balises suivent la même structure que pour les listes ordonnées, à cela près que la liste est contenue dans l'élément <ol> (ordered list), et non dans <ul>
les listes descriptives
L'objectif de ces listes est de marquer une série d'éléments et leurs descriptions associées, comme termes et définition, ou bien questions et réponse. Les listes descriptives utilisent une enveloppe de balisage différente de celle des autres types de listes — <dl> ; chaque terme est en plus entouré d'un élément <dt> (description term) et chaque description d'un élément <dd> (description definition).

Les tableaux

Une tâche assez courante en HTML consiste à structurer des données sous forme de tableaux. HTML dispose d'un certain nombre d'éléments avec attributs à cet effet. Couplé avec un peu de CSS pour styliser ces tableaux, HTML facilite l'affichage d'informations dans des tableaux sur le web.

Le style, par les CSS

Les Cascading StyleSheets — ou CSS — (Feuilles de style en cascade) sont la première technique à apprendre après le HTML. Alors que HTML s'utilise pour définir la structure et la sémantique du contenu, les CSS sont employées pour composer et déterminer l'apparence de ce contenu. Ainsi par exemple, vous utiliserez les CSS pour modifier les polices, la couleur, la taille et l'espacement de votre contenu, pour le répartir sur plusieurs colonnes ou bien pour ajouter des animations et autres fonctionnalités décoratives.

La syntaxe CSS

CSS est un langage basé sur des règles — on définit des règles de styles destinées à des éléments ou des groupes d'éléments particuliers dans la page. Par exemple, « Je veux que le titre principal de ma page s'affiche en rouge en gros caractères. » se traduira par : 


h1 {
  color: red;
  font-size: 5em;
}

La règle commence par un sélecteur, l'élément HTML mis en forme. Ici, le style s'applique aux titres de niveau 1.

Suivent une paire d'accolades { } à l'intérieur desquelles on trouve une ou plusieurs déclarations, sous la forme d'une paire propriété: valeur. Chaque paire précise une propriété de l'élément sélectionné, suivie de la valeur choisie pour cette propriété ; la propriété et la valeur sont séparées par deux points. Chaque déclaration se termine par un point-virgule. À chaque propriété définie par CSS correspondent différentes valeurs possibles.

Les sélecteurs CSS

les sélecteurs ont différentes façons de cibler le document HTML — on peut par exemple cibler les éléments selon leu type (h1) , leur classe (.special) ou leur identifiant (#monIdentifiant)

Il est possible d'appliquer une même règle à plusieurs sélecteurs regroupés en liste (utilisation de la virgule). 


      h1, .special {
        color: blue;
      }

On peut cibler des éléments en fonction de la présence d'un attribut donné pour un élément donné 


    a[title] {
      ...
    }

On peut baser la sélection sur la présence d'un attribut avec une valeur bien précise 


  a[href="https://example.com"]{
    ...
  }

Les pseudo-classes, qui ciblent des éléments dans un état donné. Par exemple, la pseudo-classe :hover sélectionne un élément seulement s'il est survolé par le pointeur de la souris 


  a:hover {
    ...
  }

on combine les sélecteurs pour cibler plus finement les éléments dans nos documents. L'exemple suivant sélectionne les paragraphes enfants directs de <article> grâce au combinateur enfant (>) 


  article > p {
    ...
  }
